## Context Notes

### Design Decisions (locked in planning session)

**DD-1: PathTierBonuses as Named Struct**
Tier bonuses use a `[Serializable]` struct with named fields (not a raw `float[]` array indexed by StatType). Safer in Inspector, matches design doc field names exactly. PathData provides `GetStatBonusArray(int tier)` to convert to the `float[]` that `StatModifierInput.pathBonuses` expects.

**DD-2: T3 Marked via [Header], No Runtime Bool**
`[Header("Tier 3 — Main Path Only")]` above T3 fields in the SO. No `bool isMainExclusive`. PathSystem (T018) enforces the constraint in behavior. All 12 path T3s are Main-only by design — a flag would always be `true`.

**DD-3: Incremental Tier Bonuses**
Each tier struct stores the *delta that tier adds*. `GetStatBonusArray(tier)` accumulates: tier 2 = T1 + T2 increments. Matches how CHARACTER-ARCHETYPES.md lists values.

### IPathProvider Update — Pre-Approved
`IPathProvider.cs` has `// TODO: Replace with PathData when T008 lands` comments on both `MainPath` and `SecondaryPath`. This swap was pre-agreed in T001. No cross-dev review needed — it resolves the TODO rather than introducing a new interface change.

### Asset Creation Strategy
A `[MenuItem("TomatoFighters/Create All Path Assets")]` editor script will create all 12 `.asset` files programmatically with the correct values. Developer runs it once in the Unity Editor. All values are sourced from CHARACTER-ARCHETYPES.md and documented in the task spec stat tables.

### Path lives in Shared/Data, not Paths/
`PathData.cs` must live in `Assets/Scripts/Shared/Data/` (namespace `TomatoFighters.Shared.Data`) so it can be referenced by `IPathProvider` (a shared interface used by all 3 pillars). The system-overview `Paths/PathData.cs` reference is outdated.
